### UC-2: Agent Interaction — Click Agent ❌ FAIL (BUG-005 NOT FIXED)
**Steps tested:**
1. Click ApoMac agent tile on Mission Desk
2. `handleAgentClick('ceo')` calls `window.openMissionControl()`
3. MC overlay gets class `visible` added
4. **MC overlay stays display:none** — CSS conflict

**Root Cause:** The overlay stacking fix (BUG-001) added:
```css
.mc-overlay { display: none !important; }
.mc-overlay.open { display: block !important; }
```
But `openMissionControl()` in `mc-core.js` adds class **`visible`**, not **`open`**.
The `.mc-overlay.visible { display: flex; }` rule is overridden by the `!important` on `.mc-overlay { display: none !important; }`.

**Impact:** 🔴 CRITICAL — Agent tiles are non-functional. Clicking any agent does nothing visible.
**Fix needed:** Either change `openMissionControl()` to add `open` class, or add `.mc-overlay.visible { display: flex !important; }` to the stacking CSS.

## Open Bugs

### 🔴 BUG-005 (STILL OPEN): Mission Control overlay blocked by CSS `!important`
- **Root cause:** `openMissionControl()` adds `.visible` but stacking CSS requires `.open` for `display: block !important`
- **Impact:** Agent tiles non-functional, Mission Control unreachable from Mission Desk
- **Fix:** 1-line CSS addition: `.mc-overlay.visible { display: flex !important; }`

### 🟡 NEW — OcStore 401 polling spam
- **Issue:** OcStore polls 4 API endpoints every 15s, all return 401 in demo mode
- **Impact:** Console noise, wasted requests (~16 req/min)  
- **Fix:** Stop polling after 3 consecutive 401s, show "not connected" state

### 🟡 NEW — MC chat agent bar still shows 6 template agents
- **Issue:** Mission Control chat (mc-center.js) has hardcoded `AGENT_DEFS` with all 6 agents
- **Impact:** Inconsistent with Mission Desk (which now shows only real agents)
- **Fix:** MC agent bar should also use dynamic agent list from API
