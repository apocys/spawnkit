# Skill Routing Guide

Quick reference: which skill to use when.

## Build / Code
| Situation | Skill |
|---|---|
| Default — any coding task | `deepcode-loop` |
| Purely additive, fully spec'd, 6+ files in parallel | `large-build` |
| Simple edit < 50 lines | Just edit directly |

## Research / Knowledge
| Situation | Skill |
|---|---|
| One-shot web research | `deep-research` |
| Regulatory/compliance — entity & relationship mapping | `lightrag` |
| User provides actual documents (PDFs, DOCX, etc.) — Q&A | `rag-anything` |
| User provides actual documents — need a verdict/risk score | `rag-diligence` |

## Analysis
| Situation | Skill |
|---|---|
| Investment/company evaluation (no documents) | `due-diligence` |
| Investment/company evaluation (with documents) | `rag-diligence` |
| Financial data, CSV, structured data | `data-analysis` |

## Content
| Situation | Skill |
|---|---|
| Long-form article | `journalist` |
| Strategy memo | `strategy-consultant` |
| Meeting notes | `meeting-notes` |
| Travel planning | `travel-planner` |

## Automation
| Situation | Skill |
|---|---|
| Make any software agent-controllable | `cli-anything` |
| Web scraping | `web-scraper` |
| Blog → podcast | `blog-to-podcast` |
